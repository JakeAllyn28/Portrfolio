# Crush Them All – Backend & Gameplay Systems

![Crush Them All Banner](https://crush-them-all.com/cta-large.jpg)

**Context:** Godzilab Inc  
**Stack:** Python, JavaScript, C++, web technologies  
**Focus:** Backend Systems, Gameplay Logic, Tooling  

---

## Overview

Worked on both backend systems and gameplay features for *Crush Them All*, an idle mobile game with large-scale progression systems and extremely high-value economies.

Focused on improving internal tools, optimizing gameplay calculations, and implementing alternative purchase flows.

---

## Problem

- Support tools were inefficient and slowed down player support workflows  
- Gameplay systems involved extremely large numerical values  
- Purchase calculations needed to be both accurate and performant  
- App store fees limited pricing flexibility  

---

## Backend & Tooling

- Developed and improved internal web tools for support staff  
- Streamlined workflows for interacting with player data  
- Improved efficiency of support operations and debugging  

---

## Gameplay Systems

- Implemented purchasing logic for in-game progression  
- Designed a performant system to compute maximum purchasable upgrades  

### Optimization Approach

- Used binary search to efficiently determine purchasable quantities  
- Avoided linear iteration due to extremely large value ranges  
- Ensured the system could run continuously (per-frame updates) without performance degradation  

---

## Payment Integration

- Implemented PayPal-based purchase flow outside of app stores  
- Enabled lower transaction fees compared to platform providers  
- Provided more flexible pricing options for players  

---

## Results / Impact

- Improved efficiency of internal support tools  
- Reduced computational cost of gameplay calculations  
- Enabled alternative monetization paths with lower overhead  
- Contributed to both backend infrastructure and core gameplay systems
