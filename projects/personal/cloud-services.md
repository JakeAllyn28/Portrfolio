# Cloud Services & Multiplayer Infrastructure

**Context:** Personal Project  
**Stack:** Unity Cloud Services, Steamworks  
**Focus:** Backend Systems, Multiplayer Infrastructure  

---

## Overview

Developed backend systems for player progression, inventory management, and multiplayer connectivity using Unity Cloud Services and Steam integration.

---

## Problem

- Needed persistent player data across sessions  
- Required matchmaking and social systems  
- Inventory and customization systems needed to be flexible and scalable  

---

## Systems Implemented

### Player Progression

- Cloud-based storage for player state  
- Persistent progression tracking  

---

### Multiplayer

- Peer-to-peer networking using Steam  
- Matchmaking and friend-based connections  
- Session management and player synchronization  

---

### Inventory & Customization

- Modular inventory system  
- Support for multiple item types and configurations  

---

## “Rune Compiler” System

Designed a flexible system for generating abilities:

- Base ability + modifiers (“runes”)  
- Combined into a final compiled result  
- Applied to spells, weapons, and other systems  

---

## Technical Details

- Data-driven system design for inventory and abilities  
- Integration between cloud storage and gameplay systems  
- Modular architecture for extensibility  

---

## Results / Impact

- Enabled scalable player progression and customization  
- Demonstrated full-stack integration (client + backend)  
- Provided reusable systems for future projects
