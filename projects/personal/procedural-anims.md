# Procedural Animation System (First-Person)

**Context:** Personal Project  
**Stack:** Unity, C#, IK systems  
**Focus:** Procedural Animation, Gameplay Systems  

---

## Overview

Developed a fully procedural animation system for first-person melee and ranged combat, eliminating the need for authored animations.

Animations were generated dynamically based on gameplay state and input.

---

## Problem

- Traditional animation pipelines require large numbers of authored assets  
- Difficult to adapt animations to dynamic gameplay situations  
- Needed consistent behavior across player and AI systems  

---

## Approach

Built a procedural animation system composed of:

- Multi-stage motion breakdown (wind-up, strike, recovery)  
- Parameter-driven movement generation  
- Real-time adjustment based on gameplay context  

---

## Key Features

- Fully procedural melee and ranged animations  
- Shared system for both player and AI  
- Dynamic response to gameplay inputs  

---

## IK Integration

- Used inverse kinematics to refine motion  
- Ensured believable positioning and alignment  
- Improved visual quality without authored animations  

---

## Technical Details

- State-driven animation generation  
- Parameterized motion curves  
- Integration with gameplay systems and hit detection  

---

## Results / Impact

- Eliminated dependency on pre-authored animations  
- Enabled flexible and scalable animation behavior  
- Demonstrated viability of procedural animation systems
